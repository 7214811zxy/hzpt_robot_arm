# 项目名称：俄罗斯方块机器人

**项目内容：**在拾取区上随机摆放7种，共35个俄罗斯方块。方块不重叠、不相接。通过一个俯拍的摄像头，我们可以识别出每个俄罗斯方块的类型、以及将其转正需要旋转的角度。所有俄罗斯会使用吸盘机械臂将其放置到14*10的盘面中。

**系统架构：**摄像头连接到PC，PC上部署了Python脚本，脚本会完成整个拾取区俄罗斯方块的识别和位姿计算。Python脚本与机械臂通过TCP Socket连接。机械臂上运行了一个自定义的lua脚本(lua_http_control.lua)，用于接收Python发送过来的机械臂需要执行的joint运动，这是一个6轴的机械臂。lua脚本中有一些机械臂厂家自定义的函数，你可能无法看到其源代码。例如table.insert(raw_pose_queue, pose)中的table变量。这是一个机械臂系统内已定义的全局变量，用户只能通过对外的insert将数值写入其中。以及set_global_variable("P001", table.unpack(current_poses.prep_pose))方法，用户只能通过该方法来修改全局变量P001的值，机械臂在特定模式下会定时访问P001储存的值，并控制机械臂运动到该位置。机械臂的负载端有一个吸盘工具，用于控制俄罗斯方块的吸取和释放。这个吸盘工具是由一个stm32控制的，stm32通过串口和PC端相连，Python使用串口向其发送0，1来控制吸合。整个系统是一个涉及到多个组件的异步通讯系统，因此联调会比较复杂。

**项目背景**：lua_http_control.lua是部署在机械臂上的lua脚本；/src/test_img/test111.jpg是一张用于测试图像识别和姿态解算算法的测试图片；/src/main.py是目前主函数的入口，但是现在是个半成品。你可以重新创建一个主函数。目前图像的识别和位姿解算功能已经完成，stats、number_list、center_list、type_list、angle_list。这几个变量共同组成了解集，这组解集目前经过测试基本可靠，因此这个求解的部分我们可以先不进行大范围的修改。现在项目的主要问题在于机械臂运动和吸盘的配合上，你可以看到我创建了一个TEST_DATA变量，这是一组已经解算完成的机械臂运动序列，绕过了求解过程，用于更方便的完成机械臂运动部分的开发，但是这块目前还存在一些问题。
